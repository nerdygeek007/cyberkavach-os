1. Background & Context
CyberKavach Club is a student-run technical and cybersecurity community. As the club has grown in membership and activity, the operational complexity has scaled in tandem. Currently, every workflow — approvals, event registrations, certificate issuance, attendance tracking, content approvals, and coordinator management — runs through a fragmented mix of WhatsApp groups, Google Forms, Excel sheets, and informal personal communication.

This fragmentation produces systemic problems:
Approval delays due to unclear escalation chains and missed messages
Duplicate event registrations and inconsistent participant records
Manual certificate generation that takes days for large cohorts
No visibility into approval status for members or coordinators
No formal recognition or reward system for volunteer contributions
Coordinators across technical, content, and social media verticals work in silos
Students have no centralized portal to register for events, form teams, or track their history

2. Problem Statement

--- Table ---
Design and develop a centralized, role-based web platform that serves as the complete digital operating system for CyberKavach Club — enabling streamlined approvals, event management, certificate automation, live attendance tracking, multi-coordinator workflows, and a member recognition system, all accessible through role-specific dashboards with real-time visibility.
--- End Table ---


3. Objectives
Deliver a secure, role-gated login system with distinct dashboards per user type.
Implement a structured multi-level approval workflow that mirrors actual club hierarchy.
Automate bulk certificate generation to eliminate manual effort for 300+ participants.
Provide a live event check-in/check-out system with real-time attendance dashboards.
Build a team-based event registration system with leader-member management.
Create a gamified appreciation and reward point system to recognize contributions.
Offer meaningful analytics and reporting for faculty and student coordinators.
Ensure the platform is scalable, maintainable, and extensible for future club needs.

5. System Modules


--- Table ---
MODULE 1 — Role-Based Authentication & Multi-Level Approval Workflow       (36 points)
--- End Table ---


5.1.1 Secure Authentication
Login system supporting all seven roles with role-based dashboard routing
First-time users can self-register; accounts require coordinator approval before activation
JWT-based session management with token refresh and secure logout
Password reset via email OTP verification
Optional: SSO integration with institutional email (Google OAuth)         (extra 3 points)

5.1.2 Permission Request System
Users can raise the following request types through the platform:
Event Permission Request — for organizing or conducting events
Resource/Venue Access Request — booking classrooms, equipment, or lab access
Budget Approval Request — financial requirements for events or activities
Social Media Posting Approval — campaign posts, announcements
Content Publishing Approval — articles, newsletters, write-ups
Certificate Generation Authorization — bulk issuance permission
External Collaboration Request — guest speakers, partner clubs          (extra 2 points)
 
5.1.3 Live Request Tracking & Timeline
Every request shows a status badge: Pending / Under Review / Approved / Rejected
Full approval timeline: who acted, at what stage, with timestamp
In-app notifications and optional email alerts on status changes
Coordinators can add remarks/comments at each approval step
Escalation alerts if a request is idle beyond a configurable threshold (e.g., 48 hours)
Make sure This process should only be visible to the person who submitted the approval request and to the Student Coordinator and Faculty Coordinator. Other coordinators and club members should not be able to view it.

MODULE 2 — Smart Certificate Generation System        (36 points)
--- End Table ---


5.2.1 Template & Participant Management
Upload and preview multiple certificate templates (PNG/PDF)
Import participant data via Excel/CSV with field mapping UI
Fields: Participant Name, Event Name ,Certificate ID
Data validation layer — flag missing or malformed entries before generation

5.2.2 Bulk Generation & Export
Auto-generate 300+ certificates in one job with dynamic text placement
Progress tracker displayed during generation

Auto-upload to Google Drive folder with shareable link
Email certificates directly to participants from within the platform (optional)      (extra 5 points)

5.2.3 Certificate Verification System
Unique auto-generated Certificate ID per certificate
Verification page confirms: Name, Event, Date, Issuing Authority
Tamper detection — verification fails if certificate has been modified
Authorized to Generate: Student Coordinator, Social Media Coordinator (with Student Coordinator approval).

Filterable participant/team table with export to CSV/Excel
Event analytics: registrations over time, team size distribution, waitlist status

Deployment | Vercel / Railway / Render + Docker | CI/CD pipeline; containerized deployment recommended

9. Constraints & Assumptions
The platform is a web-based application accessible via modern browsers on desktop and mobile.
Role assignments are managed by Faculty Coordinator and Student Coordinator.
Certificate templates are uploaded as PNG or PDF by authorized coordinators.
The system does not integrate with external payment gateways in the base version.
All data is scoped to CyberKavach Club — multi-club support is optional/bonus.
Email delivery assumes a configured SMTP server or third-party email API.
